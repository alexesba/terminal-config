# terminal-emulators/

Tracked templates for terminal emulator configs. When you pick Alacritty, Kitty,
or WezTerm in `install.sh`, it installs the app via Homebrew or your Linux
package manager if missing, then copies the chosen `.example` file into
`~/.config/` (real file, not a symlink) so Gogh and other tools can write theme
data alongside your config.

Templates use a `{{FONT_FAMILY}}` placeholder; `install.sh` replaces it with
your Nerd Font choice (default: Caskaydia Cove Nerd Font Propo) when the config
is first created.

| Template | Installed to |
|---|---|
| `alacritty.toml.example` | `~/.config/alacritty/alacritty.toml` |
| `kitty.conf.example` | `~/.config/kitty/kitty.conf` |
| `wezterm.lua.example` | `~/.config/wezterm/wezterm.lua` |

WezTerm also uses `~/.config/wezterm/colors.lua` (generated by `colorscheme`).

`update.sh` copies all three templates the first time each path is missing, so you
can theme Kitty or Alacritty later even if `install.sh` only configured WezTerm.
Use **`use-terminal`** in your shell to pick which emulator `colorscheme` targets
for the current session (see main `README.md`).

Run `./update.sh` to migrate legacy dotfiles symlinks to local copies.
